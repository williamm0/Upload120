How it works

Multiple local methods, one private file flow.

Upload120 inspects the MP4 structure, estimates video FPS from sample timing, and writes the selected container-level recipe in the browser. It does not create new frames or upload the source file anywhere.

timing + elst

Balanced Sync

Changes the main timing headers, then adds a speed guard. Use this first when you want a stronger patch but do not want the old desktop slowdown behavior.

elst + tags

Extension Signal

Adds a normal-speed edit list and iTunes-style metadata. It does not change playback speed, so the file should preview normally on a computer.

mvhd

Header Lite

Changes only the outer movie timing. It is easier on desktop players than the old method, but the upload signal is also lighter.

mdhd

Classic Force

Changes both movie and video track timing like the original open-source patchers. It can be useful as a fallback, but computer playback can look slow.
